# Jutjubic - YouTube Clone

A full-stack YouTube clone built with Spring Boot and React, featuring video uploading, real-time watch parties, JWT authentication, and more.

## Tech Stack

### Backend
- **Java 17** with **Spring Boot 3.5.7**
- **PostgreSQL** - relational database
- **Redis** - caching (thumbnails) and rate limiting
- **RabbitMQ** - message broker for upload event processing
- **Spring Security + JWT** (HS512) - authentication & authorization
- **Spring WebSocket + STOMP** - real-time watch party feature
- **Protocol Buffers** - binary message serialization for RabbitMQ
- **Springdoc OpenAPI** - Swagger UI API documentation
- **Maven** - build tool

### Frontend
- **React 19** with **TypeScript**
- **Vite 7** - build tool & dev server
- **React Router DOM** - client-side routing
- **Axios** - HTTP client
- **STOMP.js + SockJS** - WebSocket client for watch parties

### Microservices
- **RabbitMQ Consumer** - separate Spring Boot service that processes video upload events from message queues

## Prerequisites

Make sure the following are installed and running:

| Dependency   | Version | Default Port |
|-------------|---------|-------------|
| JDK         | 17+     | -           |
| Node.js     | 16+     | -           |
| PostgreSQL  | 12+     | 5432        |
| Redis       | 6+      | 6379        |
| RabbitMQ    | 3.x     | 5672        |

## Database Setup

Create a PostgreSQL database named `jutjubic`:

```sql
CREATE DATABASE jutjubic;
```

The schema is auto-generated by Hibernate on startup, and sample data is loaded from `import.sql`.

## Getting Started

### 1. Clone the repository

```bash
git clone https://github.com/darjanrac/youtube-clone.git
cd youtube-clone
```

### 2. Configure the backend

Edit `jutjubic-be/src/main/resources/application.properties` if your local setup differs from the defaults:

```properties
spring.datasource.url=jdbc:postgresql://localhost:5432/jutjubic
spring.datasource.username=postgres
spring.datasource.password=your_password

spring.data.redis.host=localhost
spring.data.redis.port=6379

spring.rabbitmq.host=localhost
spring.rabbitmq.port=5672
```

### 3. Start the backend

```bash
cd jutjubic-be
./mvnw spring-boot:run
```

The API starts on **http://localhost:8084**.

### 4. Start the frontend

```bash
cd jutjubic-fe
npm install
npm run dev
```

The app opens on **http://localhost:5173**.

### 5. (Optional) Start the RabbitMQ consumer

```bash
cd rabbitmq-consumer-example
./mvnw spring-boot:run
```

## API Endpoints

### Authentication
| Method | Endpoint              | Auth | Description              |
|--------|-----------------------|------|--------------------------|
| POST   | `/api/auth/register`  | No   | Register a new account   |
| GET    | `/api/auth/activate`  | No   | Activate account (email) |
| POST   | `/api/auth/login`     | No   | Login, returns JWT token |

### Videos
| Method | Endpoint                        | Auth | Description             |
|--------|---------------------------------|------|-------------------------|
| GET    | `/api/videos`                   | No   | List all videos (paged) |
| GET    | `/api/videos/search`            | No   | Search videos           |
| GET    | `/api/videos/{id}`              | No   | Get video details       |
| POST   | `/api/videos/upload`            | Yes  | Upload a video          |
| POST   | `/api/videos/{id}/view`         | No   | Increment view count    |
| GET    | `/api/videos/{id}/thumbnail`    | No   | Get cached thumbnail    |
| GET    | `/api/videos/{id}/comments`     | No   | Get video comments      |

### Comments
| Method | Endpoint          | Auth | Description        |
|--------|-------------------|------|--------------------|
| POST   | `/api/comments`   | Yes  | Post a comment     |

### Users
| Method | Endpoint          | Auth | Description        |
|--------|-------------------|------|--------------------|
| GET    | `/api/users/{id}` | No   | Get user profile   |

### Watch Party
| Method | Endpoint                         | Auth | Description          |
|--------|----------------------------------|------|----------------------|
| POST   | `/api/watchparty/create`         | Yes  | Create a room        |
| POST   | `/api/watchparty/join/{roomId}`  | Yes  | Join an existing room|

WebSocket endpoint: `ws://localhost:8084/ws` (SockJS + STOMP)

## Features

- **User registration & login** with email activation and JWT-based sessions
- **Video upload** with thumbnail support (max 210 MB per file)
- **Video browsing & search** with pagination
- **Comments** on videos
- **View counting**
- **Watch Party** - synchronized video playback with other users via WebSocket
- **Rate limiting** - Redis-backed login throttling (5 attempts / 60 seconds)
- **Thumbnail caching** via Redis
- **Event-driven architecture** - upload events published to RabbitMQ in both JSON and Protobuf formats
- **Swagger UI** for interactive API exploration
